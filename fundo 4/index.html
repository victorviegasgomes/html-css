Para criar *apps mobile*, o número de *linguagens de programação* que você precisará aprender depende de vários fatores, como a plataforma alvo (Android ou iOS), o tipo de app (nativo, híbrido ou multiplataforma) e as suas preferências pessoais (se você quer usar ferramentas específicas, por exemplo).
Vou explicar os cenários comuns e as linguagens associadas a cada um.

1. *Apps Nativos* (Android e iOS)

Android:

- *Linguagens principais*:

- *Java*: A principal linguagem usada no desenvolvimento de apps Android até alguns anos atrás. Ela ainda é amplamente usada e muitas bibliotecas e frameworks são feitos em Java.

- *Kotlin*: Atualmente, é a linguagem preferida para o desenvolvimento Android, pois é mais moderna e oferece muitos recursos avançados, com menos código e mais segurança em comparação com o Java. O Kotlin foi oficialmente apoiado pelo Google em 2017.

Portanto, para *Android*, você pode usar *Java* ou *Kotlin*.

iOS:

- *Linguagens principais*:

- *Objective-C*: Foi a principal linguagem para iOS antes do Swift. Embora ainda seja usado em muitos projetos legados, não é mais a escolha moderna para o desenvolvimento de apps.

- *Swift*: Atualmente, é a linguagem preferida para iOS. Foi criada pela Apple para ser segura, rápida e mais fácil de usar do que o Objective-C. O Swift é amplamente adotado por desenvolvedores de iOS e é suportado por todos os recursos mais recentes do sistema operacional da Apple.

Portanto, para *iOS*, você pode usar *Objective-C* ou *Swift*.

Conclusão:

Para desenvolvimento *nativo* de apps, você precisará aprender:

- *Android*: *Java* ou *Kotlin*.

- *iOS*: *Objective-C* ou *Swift*.

2. *Desenvolvimento Multiplataforma*

Se você deseja criar um *app* que funcione tanto no *Android* quanto no *iOS* com o mínimo de código duplicado, você pode optar por *ferramentas multiplataforma*. As linguagens e frameworks mais comuns incluem:

a) *Flutter*:

- *Linguagem*: *Dart*.

- *Descrição*: O Flutter é um framework desenvolvido pelo Google que permite criar apps nativos para Android e iOS com o mesmo código base. Ele é baseado na linguagem *Dart*, que foi criada pelo Google. É muito popular por sua performance rápida e a aparência nativa.

b) *React Native*:

- *Linguagem*: *JavaScript*.

- *Descrição*: React Native é um framework baseado em *JavaScript* e *React* (uma biblioteca JavaScript para criar interfaces de usuário). Ele permite escrever código que pode ser executado em ambos Android e iOS, embora seja possível escrever partes do código em *Swift* (para iOS) ou *Java*/*Kotlin* (para Android) quando necessário.

c) *Xamarin*:

- *Linguagem*: *C#*.

- *Descrição*: O Xamarin é uma plataforma de desenvolvimento multiplataforma que usa *C#* para criar apps nativos para iOS, Android e Windows. Ele oferece uma boa integração com os sistemas nativos e usa um único código base.

d) *Ionic*:

- *Linguagem*: *JavaScript* (usando *Angular* ou *React*).

- *Descrição*: O Ionic é um framework que usa *HTML*, *CSS* e *JavaScript* para criar apps híbridos, ou seja, apps que são executados em um WebView (um navegador incorporado) dentro do app. Ele pode ser usado para desenvolver apps para Android e iOS com o mesmo código.

Conclusão:

Para *desenvolvimento multiplataforma*, as principais linguagens são:

- *Flutter*: *Dart*.

- *React Native*: *JavaScript*.

- *Xamarin*: *C#*.

- *Ionic*: *JavaScript*.
